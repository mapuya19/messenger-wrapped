# AGENTS.md - Project Guide for AI Assistants

## Project Overview

**Messenger Wrapped** - A privacy-first web app that generates Spotify Wrapped-style analytics from Facebook Messenger chat data. All processing happens locally in the browser.

**Stack**: Vite + React + TypeScript + TanStack Router + Tailwind CSS

## Architecture

### File Structure
```
src/
├── main.tsx              # Entry point - imports index.css here
├── index.css             # Tailwind directives + global styles
├── routes/               # TanStack file-based routing
│   ├── index.tsx         # Home page
│   └── __root.tsx        # Root layout (wraps all routes)
└── routeTree.gen.ts      # Auto-generated by TanStack Router

components/               # React components organized by feature
├── dashboard/            # Stats dashboard
├── story/                # Story slideshow slides
├── ui/                   # Shared UI primitives
└── upload/               # File upload components

lib/
├── analyzer/             # Data analysis logic
├── parser/               # Messenger JSON parser
└── utils/                # Helper functions
```

### Key Configuration Files

- **`vite.config.ts`** - Vite build config (includes `@tailwindcss/vite` plugin)
- **`src/index.css`** - Tailwind entry point; custom theme defined here via `@theme {}`
- **`tsr.config.json`** - TanStack Router config (file-based routing)

## Common Tasks

### Adding a New Route
```bash
npm run gen  # Regenerate route tree after adding files to src/routes/
```

Routes are file-based:
- `src/routes/index.tsx` → `/`
- `src/routes/about.tsx` → `/about`
- `src/routes/blog/$postId.tsx` → `/blog/:postId`

### Styling
- Uses Tailwind CSS utility classes
- Custom CSS variables in `src/index.css` (e.g., `--messenger-blue`)
- Dark theme by default (background: `--messenger-dark`)

### State Management
- Global state: `ChatDataContext` in `contexts/ChatDataContext.tsx`
- Route state: TanStack Router's built-in state management

## Build & Deploy

```bash
npm run dev      # Development server on port 3000
npm run build    # Production build to /dist
npm start        # Preview production build
```

**Deployment**: Configured for Vercel (`vercel.json` present)

## Important Notes

1. **Never remove the CSS import** in `src/main.tsx` - Tailwind won't work without it
2. **Always run `npm run gen`** after adding/modifying route files
3. **Custom theme tokens** live in `@theme {}` in `src/index.css` (not `tailwind.config.js` — that file no longer exists)
4. **No server-side code** - this is a static SPA, all data processing is client-side
5. **Privacy is critical** - never add analytics, tracking, or data transmission

## Dependencies to Know

- `@tanstack/react-router` - File-based routing
- `motion` - Animations for story slides (`import { motion } from 'motion/react'`)
- `html-to-image` - Download wrapped as images
- `recharts` - Charts for dashboard
- `jszip` - Unzip Messenger exports
